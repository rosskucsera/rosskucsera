Expensify Usability Exercise

Overview

I chose to rework a single interaction within a web app called Expensify used to track company expenses. I thought there was an opportunity improve how Expensify gives you feedback as you are going through this process.

Problem

When I use Expensify I am generally switching back and forth between different browser tabs to download various receipts for uploading that it's easy to lose track of what row you were working on last.

Solution

I created a new section below the main list of expenses to visually separate the two. Now instead of hunting around for the small receipt icon to see if that receipt is ready to submit, the distinction is made more clear.

Skills used

  • Research
  • HTML/CSS

What I always nd frustrating is that it's not easy to visually see which expenses are done and which ones I still need to add a receipt for.

I created a new section below the main list of expenses to visually separate the two. Now instead of hunting around for the small receipt icon to see if that receipt is ready to submit, the distinction is made more clear.

There's something satisfying about seeing the expense move down to the 'Ready to submit' section. While a simple enough change (it could be as easy as adding a new CSS class to the row element when a receipt is added to it), it actually adds a bit of excitement to the mundane task of expense management.
